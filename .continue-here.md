<decisions_made>

- Nothing like this exists in the .NET ecosystem — confirmed novel
- Skills are bidirectional rules (describe/scan/generate/verify), not just verification scripts
- Rules decompose into decision trees: L1 mechanical → L5 creative. Scripts handle L1-L3, bubble L4+ back as structured questions
- Per-rule configurable heuristic thresholds (Act/Suggest/Ask bands)
- Three-layer learning: (A) refine script, (B) skill-level memory/pitfall, (C) human feedback — all require operator confirmation
- Sub-script granularity: atomicity primary, one action per leaf, one file per leaf
- V1 never guesses at L4 — returns the question with context and options
- Model integration deferred to V2 (bubble to calling agent for now)

</decisions_made>
